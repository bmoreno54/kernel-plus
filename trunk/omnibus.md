# Omnibus Tag Substrate — Emergent Semantic Coordination

> **scope:** trunk — inherited by all branches
> **status:** active

## What the Omnibus Is

The omnibus is a coordination substrate — a shared medium through which
agents (human and AI, present and future) communicate about the structure
of the ecology without needing to agree on a schema in advance. Tags carry
semantics. Folders carry files. Structure emerges from use, not planning.

Each tag application is a local action (one agent, one moment, one block
of text). The tag topology is a global property — it connects surfaces
that were never intended to be connected, reveals clusters that no one
planned, and enables queries no folder tree could answer. This is
stigmergic coordination: each tagged surface is a pheromone that directs
future agents without any direct communication channel.

## Why Tags Over Folders

A taxonomy forces every item into one path in a tree. A tag system allows
every item to participate in multiple overlapping sets. The combinatorial
space of tag intersections is exponentially richer than any tree can express.

Adding a new tag costs nothing and retroactively enriches every item that
shares tag-space with it. Tags are duck-typed: a block tagged `#card` acts
as a card when queried as one, regardless of what file it lives in.

Folder structure remains useful as a spatial metaphor for browsing. But the
semantic load shifts to tags. Refactoring folders doesn't break semantic
organization because tags travel with content.

## The Tag Taxonomy

### Universal Signal

- `#omnibus` — meta-tag signaling "this belongs in the substrate." A
  breadcrumb. Appears anywhere. If you see `#omnibus`, it's a surface
  that hasn't been fully indexed yet.

### Density Layer Tags (Penumbra Pattern)

- `#card` — state snapshot, 10-30 lines, answers "what's the situation?"
- `#grip` — configuration surface, tuning lever, point where intention
  grabs the system
- `#deep` — full reference, engage only when penumbra doesn't resolve

### Semantic Role Tags

- `#research` — exploratory, may change, not yet actionable
- `#infrastructure` — operational, supports the project's own processes
- `#library` — archived for study, not for active build
- `#output` — deliverable, artifact, build product
- `#state` — snapshot of current conditions, decays quickly
- `#decision` — a choice point that gates downstream work

### Momentum Tags

- `#blocked` — can't activate until something else resolves
- `#tested` — confirmed working by direct observation
- `#untested` — theorized but not verified

### Kernel+ Tags

- `#kernel+` — marks an atom as domain-independent (passes the veil of
  ignorance test). Used during FADE to self-identify generalizable insights.

### Attractor Tags

- `#attractor` — marks a concept functioning as a basin of attraction
- `#nucleation` — marks a document that crystallized from penumbral accretion
- `#dark` — marks an inferred pattern not yet explicit in any single document

### Branch-Specific Extensions

Branches extend the taxonomy with domain-specific tags. These live in the
branch, not the trunk. Your domains will develop their own tag vocabulary
through use.

## Praxis

1. **Tag in situ.** When you encounter a surface during normal work, tag
   it inline. Don't break flow.
2. **Promote on revisit.** When you revisit a doc and see `#omnibus`
   breadcrumbs, promote to full entries.
3. **Diagram causal chains.** When a `#procedural` sequence spans 3+
   surfaces, create a diagram.
4. **Node docs are living.** They grow as surfaces are discovered. Never
   need to be "complete."
5. **Intersection queries.** The power is in tag intersection:
   `#automatable AND #infrastructure`, `#research AND #blocked`.

## Operational Dynamics

### Penumbral Dynamics (No Wrong Present State)

Knowledge domains have two equally viable dynamics:

**Accretion (bottom-up nucleation).** Cards and grips accumulate as
fragments during normal work. When density exceeds a threshold — when the
penumbral field becomes supersaturated — a deep doc crystallizes. The
penumbra generates the core.

**Gravity wells (top-down radiation).** A deliberate FADE investigation
produces a deep doc that then radiates penumbral material — cards, grips,
diagnostics. The core generates the penumbra.

Both happen. Neither is privileged. And there is no wrong present state:
a penumbral cloud with no center is fine (tags make it queryable). A deep
doc with sparse penumbra is fine (recently crystallized). A domain with
neither is fine (pre-signal).

### Intersubjective Constitution

Documents constitute each other's meaning through shared tags. A card
tagged `#project-x #crash` doesn't have standalone semantics — its meaning
shifts when a grip map tagged `#project-x #config` is added. No document
was edited, but every document was changed.

Tags are relational, not categorical. When you query a tag, you get a
phenomenological field — perspectives that collectively constitute the
domain — not a flat list of co-located files.

### Strange Attractors (Dark Patterns)

Tag clusters reveal implicit organizational forces — basins in the
project's phase space toward which work naturally converges without anyone
planning it. These are "dark" in the astrophysical sense: gravitational
pull on adjacent decisions without being directly visible.

When you notice multiple unrelated discoveries clustering around a shared
property, that clustering IS the dark pattern. Name it. Track it. But
don't flatten it into a rigid category — an attractor is a tendency, not
a rule.

## Big Tags (Ritualized Axioms)

Some positions compress into aphoristic phrases carrying more gravitational
weight than any hashtag. These are big tags — philosophical touchstones
that reorient whatever they touch without re-derivation.

Big tags accrete as YOU articulate them. They are not invented by agents.
They ritualize through repeated invocation. When one appears in a document,
it snaps surrounding material back to that axiom.

## Compressed Procedural Triggers

As you work, you'll develop shorthand — compressed notation that encodes
complex operational protocols into minimal tokens. Document these here as
they emerge. Properties:

- **Compressed:** Each trigger encodes a protocol requiring paragraph-length
  instruction otherwise
- **Ritualized:** Accreted through your use, not designed top-down
- **Stigmergic:** A trigger deposited in conversation alters agent behavior
  like a pheromone — without explicit communication
- **Ecology-scale:** Any session should recognize and respond to your triggers

## Hypervisory Use

The omnibus enables detection of cross-branch patterns. When the same tag
intersection appears in multiple branches, that's a signal: a shared
concern wants to condense. The omnibus is the substrate that makes
opportunistic separation safe — tags travel with content, so splitting a
project doesn't sever semantic connections.

## Kernel+ Dimension

The kernel+ dimension runs parallel to all FADE work. When FADE decomposes
an atom that feels domain-independent, tag it `#kernel+` inline, right
where it sits. No upward deposit. No transport. Pheromones in place,
harvested periodically with judgment.

The veil of ignorance test: "Would this insight be useful to someone with
NO knowledge of the specific domain it came from?" If yes → `#kernel+`.
