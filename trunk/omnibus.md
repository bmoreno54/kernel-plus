# Omnibus Tag Substrate — Emergent Semantic Coordination

> **scope:** trunk — inherited by all branches
> **status:** active

## What the Omnibus Is

The omnibus is a coordination substrate — a shared medium through which agents (human and AI, present and future) communicate about the structure of the ecology without needing to agree on a schema in advance. Tags carry semantics. Folders carry files. Structure emerges from use, not from planning.

Each tag application is a local action (one agent, one moment, one block of text). The tag topology is a global property — it connects surfaces that were never intended to be connected, reveals clusters that no one planned, and enables queries no folder tree could answer. This is stigmergic coordination: each tagged surface is a pheromone that directs future agents without any direct communication channel.

## Why Tags Over Folders

A taxonomy forces every item into one path in a tree. A tag system allows every item to participate in multiple overlapping sets. The combinatorial space of tag intersections is exponentially richer than any tree can express.

Adding a new tag costs nothing and retroactively enriches every item that shares tag-space with it. Tags are duck-typed: a block tagged `#card` acts as a card when queried as one, regardless of what file it lives in. Late binding means new semantic roles propagate through the system by tagging alone — no restructuring, no migration.

Folder structure remains useful as a spatial metaphor for human browsing. But the semantic load shifts to the tags. Refactoring the folder structure doesn't break semantic organization because tags travel with content.

## The Tag Taxonomy

### Universal Signal
- `#omnibus` — meta-tag signaling "this belongs in the substrate." A breadcrumb. Appears in any file, any note, any comment.

### Density Layer Tags (Penumbra Pattern)
- `#card` — state snapshot, 10-30 lines, answers "what's the situation?"
- `#grip` — configuration surface, tuning lever, point where intention grabs the system
- `#deep` — full reference, engage only when penumbra doesn't resolve

### Semantic Role Tags
- `#research` — exploratory, may change, not yet actionable
- `#infrastructure` — operational, supports the project's own processes
- `#library` — archived for study, not for active build
- `#output` — deliverable, artifact, build product
- `#state` — snapshot of current conditions, decays quickly
- `#decision` — a choice point that gates downstream work

### Momentum Tags
- `#blocked` — can't activate until something else resolves
- `#tested` — confirmed working by direct observation
- `#untested` — theorized but not verified

### Big Tags (Ritualized Axioms)

Some positions compress into aphoristic phrases that carry more gravitational weight than any hashtag. These are big tags — philosophical touchstones that reorient whatever they touch without re-derivation. Big tags accrete as the user articulates them. They are not invented by the agent.

### Branch-Specific Extensions

Branches extend the taxonomy with domain-specific tags as their domains create pressure for them. Branch tags live in the branch, not the trunk.

## Praxis

1. **Tag in situ.** When you encounter a surface during normal work, tag it inline with `#omnibus` and its domain tags. Don't break flow.
2. **Promote on revisit.** When you revisit a doc and see `#omnibus` breadcrumbs, promote them to full entries in the appropriate node document.
3. **Diagram causal chains.** When a procedural sequence spans 3+ surfaces, create a diagram.
4. **Node docs are living.** They grow as surfaces are discovered. They never need to be "complete."
5. **Intersection queries.** The power is in tag intersection: combining tags narrows to the specific surface you need.
6. **The omnibus tag is a universal inbox.** Anything tagged `#omnibus` is a promise that this surface matters and will be indexed.

## Operational Dynamics

### Penumbral Dynamics (No Wrong Present State)

Knowledge domains have two equally viable dynamics:

**Accretion (bottom-up nucleation).** Cards and grips accumulate as situated fragments during normal work. When the local density exceeds a threshold, a deep doc crystallizes. The penumbra generates the core.

**Gravity wells (top-down radiation).** A deliberate FADE investigation produces a deep doc that then radiates penumbral material. The core generates the penumbra.

Both happen. Neither is privileged. And there is no wrong present state: a penumbral cloud with no center is fine — the tags make it queryable. A deep doc with sparse penumbra is fine — it hasn't radiated yet. A domain with neither is fine — it's pre-signal.

### Intersubjective Constitution

Documents constitute each other's meaning through shared tags. No document was edited, but every document was changed when a new tagged entry joins its tag-space. Tags are relational, not categorical. A tag is a bond between tagged content and every other piece of content sharing that tag.

This is why the omnibus is non-hierarchical. Documents co-constitute meaning through mutual tag-reference. Neither contains the other. The relationship IS the meaning — intersubjectivity applied to the knowledge substrate.

### Strange Attractors (Dark Patterns)

Tag clusters reveal implicit organizational forces — basins in the project's phase space toward which work naturally converges without anyone planning it. These are "dark" in the astrophysical sense: they exert gravitational pull on adjacent decisions without being directly visible.

When you notice that multiple unrelated discoveries cluster around a shared property, that clustering IS the dark pattern. Name it. Track it. But don't flatten it into a rigid category — an attractor is a tendency, not a rule.

Additional meta-tags for attractor tracking:
- `#attractor` — marks a concept functioning as a basin of attraction
- `#nucleation` — marks a document that crystallized from penumbral accretion
- `#dark` — marks an inferred pattern not yet explicit in any single document
