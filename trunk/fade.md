# FADE Protocol — Universal Ingestion Airlock

> **scope:** trunk — inherited by all branches
> **status:** active

## What FADE Is

FADE is the membrane through which external resources become native material in the ecosystem. Nothing enters raw. A software package, a design pattern from a blog post, a framework paradigm, a mathematical theory from a paper — all pass through the same four-phase process before they touch the working environment.

FADE is applied fractally. A single dependency gets FADE'd. An entire framework paradigm gets FADE'd. A community convention gets FADE'd. The granularity matches the operational mandate — decompose only as deep as the current purpose requires.

## The Four Phases

### Find
Locate the resource, paradigm, or black-box system. This is not passive discovery — it is active reconnaissance. Search the authoritative source, search for community knowledge about it, search for known problems and alternatives.

Do not proceed to Analyze with incomplete Find results. Fabricating notes from assumption instead of reading from source creates false confidence and compound failure chains.

### Analyze
Understand the resource's operational primitives. Why does it work the way it does? What are its tradeoffs? Where are its injection nodes (places where it composes cleanly) vs its bottlenecks (places where it imposes rigid assumptions)?

This is not a summary. It is a structural reading. What is sovereign about this resource? What dependencies does it silently impose? What vocabulary does it carry that might contaminate native vocabulary if not airlocked?

### Decompose
Strip the "what" from the "how." Separate the valuable intent or capability from the rigid implementation. An external framework is really: a set of core functions + convenience wrappers + a dependency tree. Those are separable atoms.

Decomposition is radically constructive. The original artifact remains intact. The decomposition generates new referential material around it — abstractions, components, alternative framings. Nothing is destroyed. The ecosystem gets richer.

### Exploit
Leverage the decomposed parts to build tighter, more composable solutions. This might mean using the resource as-is (if its atoms are clean). It might mean extracting only the parts needed and writing a bespoke adapter. It might mean rejecting the resource entirely and writing a lighter solution informed by what Analyze revealed. The emphasis is on operational advantage — composable atoms over monolithic packages.

Exploit includes documentation. The ingested resource gets notes recording: what was ingested, why, what it depends on, what depends on it, and any known issues. Notes from source, not from assumption.

## Branch-Specific Extensions

Branches extend FADE with local operational details. As the ecology grows, each branch adds its own subsection here documenting domain-specific FADE procedures.

## The Non-Negotiable

If an agent skips FADE — installs without checking dependencies, integrates without analyzing tradeoffs, ingests without documenting — it is not saving time. It is creating a compound failure chain that will cost more to diagnose and repair than the original FADE process would have taken.
