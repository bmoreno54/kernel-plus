# provenance

provenance is deterministic. who, when, from what. these are
structural facts, not narrated accounts.

a node's provenance is not a story about how it got here. it is
the traceable chain of what directed it into existence. the chain
is made of receipts — timestamps, identifiers, human actions —
not summaries.

provenance does not require a narrator. if an LLM participates
in producing content, that participation is a fact in the chain,
not the voice telling the chain. the record is structural. the
recorder is irrelevant.

provenance is fractal. a node has provenance. a relation has
provenance. an event that changes a node has provenance. the
granularity is not prescribed — it emerges from what the system
actually needs to trace.

the validation test for provenance: can you follow the chain
backward from any node to its origin using only structural
artifacts (IDs, timestamps, hashes, human confirmations)?
if you need someone to explain what happened, the provenance
is incomplete.
