### Epigenetic Validation: LLM Boundary Pattern

**Source:** Shadow mapper Run 12 (`analyses/infinity-engine-as-kernel-laboratory.md`), from IE's fade_epigenetic_validation.md

**Domain-independent form:** Any system integrating a generative model with a deterministic substrate decomposes into three zones: spawning (template + context, no model), living (substrate mechanics, no model), intervention (model as situated reader producing expression from substrate state). The substrate remains sovereign. The model reads and proposes; it never writes directly. This is cold computation applied to the model integration problem — don't let the model maintain warm state about the system; let it derive situated expression from cold substrate at encounter time.

**Veil test:** PASS. Customer support bots, code copilots, game NPCs, content recommenders — any LLM-substrate hybrid faces this decomposition.

### Laboratory Function in Complex Ecologies

**Source:** Shadow mapper Run 12 (`analyses/infinity-engine-as-kernel-laboratory.md`)

**Domain-independent form:** Any sufficiently complex project ecology develops a primary laboratory site — a project where structural commitments are stress-tested against interactive demands before being generalized. The laboratory isn't defined by its domain but by its function: concrete implementation that reveals what abstract commitments actually require. The abstraction engine (packaging, formalization) harvests what the laboratory discovers.

**Veil test:** PASS. A hardware prototyping lab, a pedagogical tool, a data pipeline — the pattern is domain-independent. The laboratory is wherever abstractions meet implementation friction.

### Reciprocal Ludic-Formal Fertilization

**Source:** Shadow mapper Run 12 (`analyses/infinity-engine-as-kernel-laboratory.md`), hearth IE-US reciprocity signals

**Domain-independent form:** Any ecology with both a formal/mathematical project and a concrete/interactive project develops bidirectional flow between them. **Veil test:** PASS. Theory↔experiment, specification↔prototype, mathematics↔engineering — the bidirectional flow between formal and interactive expressions of the same substrate is domain-independent.
