# kernel+

A sovereign ecology seed — structural commitments and operational protocols for building a personal knowledge ecosystem that grows from your friction.

## What This Is

Five foundational positions, twelve operational protocols, seven domain templates, and 156 structural atoms. Distilled through a veil of ignorance test from a living ecology. Domain-specific content stripped. What remains is what generalizes to any human building any kind of project ecosystem.

The kernel doesn't tell you what to build. It gives you structural logic — sovereignty, composition, provenance, ingestion, separation — that holds regardless of domain. The seeds show what working looks like. The harvests give you intellectual fuel.

## Quick Start

1. Clone this repo
2. Read `ONBOARD.md` — full first-session protocol
3. Edit `alpha_entity.md` — claim it as yours
4. Pick a seed from `seeds/`, copy it, run your first FADE
5. `git commit` — the ecology is alive

## Structure

```
KERNEL.md           # canonical agent instructions (read first)
ONBOARD.md          # first-session protocol (10 min to productive)
alpha_entity.md     # you (sovereign origin, claim it)
kernel/             # foundational positions (5 documents)
trunk/              # operational protocols (12 documents)
seeds/              # domain templates (7 templates, copy and fill)
harvests/           # structural atoms (156 across 4 harvests)
notes/              # process documentation
adapters/           # platform-specific agent files
```

## Agent Independence

KERNEL.md is the canonical instruction surface. Platform-specific files (CLAUDE.md, .cursorrules, etc.) are adapter emanations. The kernel works with any agent that can read markdown.

## Register

This build is tuned for someone fluent across tech, academic, scientific, creative, political, gaming, and mystic registers. The seeds cover these domains. The harvests draw from all of them. If your bandwidth is narrower, ignore what doesn't apply — the structural logic holds regardless.

## Kernel+ Extensions

The veil of ignorance is tunable. This repo ships with aggressive preseeding across multiple registers. As you work, your ecology will develop domain-specific patterns that don't pass the veil — that's healthy. The kernel is what generalizes; your domains are what specialize.
